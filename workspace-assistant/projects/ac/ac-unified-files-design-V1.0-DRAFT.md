# Phase 16 — 统一文件管理

> 当前痛点：聊天 📎 文件选择器和 📁 文件浏览器各玩各的，上传/选择路径不统一。
> 目标：`user-files` 成为聊天和文件浏览器共享的「我的文件」中枢。

---

## 一、聊天输入区直接上传

### 1.1 拖拽上传
- 用户从桌面拖文件到聊天输入区 → 自动上传到 `/user-files/`
- 显示内联进度条（聊天输入区上方或 📎 按钮处）
- 上传成功后自动出现在附件标签中，随下一条消息发送

### 1.2 📎 按钮增强
- 点 📎 → 弹窗分两个 Tab：
  - **「我的文件」Tab**：浏览 `/user-files/` 内文件（现有功能）
  - **「上传」Tab**：点选/拖拽上传文件到 `/user-files/`，上传完自动跳到「我的文件」
- 底部始终显示「从电脑上传」按钮（`<input type="file">`），支持批量选择

### 1.3 消息附件
- 发送消息时附带已选文件路径列表 → 后端读取文件内容注入 prompt
- 对话中显示附件小标签（文件名 + 类型图标）

---

## 二、文件选择器增强

### 2.1 多目录浏览
- 文件选择器不再死绑 `/user-files/`，顶上加面包屑/下拉切换目录：
  - `📁 我的文件`（/user-files，默认）
  - `📁 项目文件`（/workspace-assistant）
  - `📁 Build喵`（/workspace-build-cat）

### 2.2 不上传 — 只选文件
- 文件选择器的职责就是「选文件」——不上传
- 上传统一走两个入口：
  - **聊天输入区**：拖拽或 📎 内「上传」Tab → 进 `/user-files/`
  - **文件浏览器**：拖拽上传区 → 进当前浏览目录
- 用户不会困惑「这个上传到底传去哪了」

---

## 三、文件浏览器「我的文件」入口

### 3.1 目录树快捷方式
- 根目录下 `user-files` 用 📂 特殊图标 + 「我的文件」标签高亮
- 始终排在目录树第一位
- 上方显示「拖拽文件到这里上传」提示条

### 3.2 上传区域增强
- 进入 user-files 目录后，上传提示改为「📤 上传到我的文件（拖拽或点击）」
- 上传完成后整棵树刷新，聊天侧的文件选择器也同步更新

---

## 四、统一数据流

```
用户动作                    存储位置              影响范围
─────────                  ────────              ────────
聊天拖拽上传          →    /user-files/   →    文件选择器 + 文件浏览器
聊天 📎 内上传        →    /user-files/   →    文件选择器 + 文件浏览器
文件浏览器上传        →    当前浏览目录    →    文件浏览器（含 user-files）
文件选择器上传        →    当前浏览目录    →    文件选择器 + 文件浏览器
```

规则：
- **聊天侧上传统一进 `/user-files/`**（不管当前在哪个 Tab 浏览了什么目录）
- **文件浏览器侧上传进当前目录**（保持现有行为）
- **文件选择器上传进当前浏览的目录**（保持灵活）
