### 输出问题审查

#### 🔴 信息错误/缺失
11. **build_task 标题缺少"部分失败"状态** — `build_task()` 第 692-695 行只有"全部完成"和"失败"两种，没有像 `_build_merged_card` 那样的"🌦 部分任务失败"（橙色）。如果多集群 build 其中一个失败，标题直接显示"🌧️ 自动构建失败"而不是"部分失败"
12. **build_task 的 cluster_results 缺少 branch 字段** — 第 610 行 fallback 的 dict 没有 `branch`，但 status_desc 里显示了 `branch`（从 params 取），如果 result 不是 dict 类型就缺了
13. **test_task 和 _build_merged_card 的 status_desc 不一致** — test_task 中有 `partial_failure` 状态，但 build 侧（build_task 第 663-673 行）没有 `partial_failure`（虽然 build 没有子测试概念，但如果 build_exports 有 error 也应该区分）
14. **单行翻译路径的展示信息比多行少** — 单行路径（`_translate_and_execute` → 第 1355-1360 行）有 `📝 已收到请求` 和 `🔄 翻译后的构建指令`，但没有 `lead_parts` 中的详细信息（集群/分支/镜像/任务 id），格式跟多行路径不一致

#### 🟡 用户理解不清
15. **"点击查看 build log"在文本模式下无法点击** — 当卡片超阈值走方式2（文本超链接）时，`_build_result_card` 会上传 log 生成 gitlab 链接。但如果上传失败，显示"log 上传失败，发送 query {task_id} 查看"——用户可能不知道怎么用 query
16. **build 缓存命中时展示的信息不够** — 缓存命中显示"📦 缓存命中（原始任务: xxx，构建时间: xxx）"，但没说用的是哪个 commit、哪个镜像构建的，用户不确定缓存的包是不是最新的
17. **test_task 的 header_text 信息量差异大** — 成功时显示"（X 组测试全部通过）"有数量，但 `_build_merged_card` 成功时只显示"自动测试全部完成"没有数量

#### 🟢 格式/细节
18. **build_task 的 header 没用 emoji 天气图案** — build_task 用"☀️"和"🌧️"，_build_merged_card 多了一个"🌦"（部分失败），但 build_task 没有
19. **超时时间从 TIMEOUT_DEFAULTS 读取的是全局默认值** — 第 1288 行 `config.TIMEOUT_DEFAULTS.get(_timeout_key, 10)`，但仓库可能有自己的 `build_timeout`/`test_timeout` 覆盖值（存在 repo_dict 里），显示的超时时间可能跟实际不一致

### 用户异常输入场景（代码无法处理或处理不好）

#### 🔴 会出错或无响应
20. **"取消/停止刚才的 build"** — 没有取消任务功能
21. **"重新跑一下上次失败的"** — 没有重试功能，NLP 难以提取精确参数
22. **"把每日任务第二条改成 xxx"** — parse_change 只支持 add/delete，不支持修改第 N 条
23. **"build 完自动测一下"** — build 和 test 无联动机制
24. **"用昨天那个环境编译"** — 模糊时间指代，NLP 无法理解
25. **"编完发给我/把 wheel 发我飞书"** — 只返回路径，不发文件

#### 🟡 能跑但用户会困惑
26. **"好了没"** — 走 NLP 翻译，不会查任务状态。应翻译为 query
27. **"所有仓库都 build"** — 组合爆炸，可能几十条指令
28. **超时时间显示与实际不一致** — 展示从 TIMEOUT_DEFAULTS 读（10分钟），实际可能 60 分钟
29. **daily 命令格式错误** — add daily 直接 append 字符串，执行时 parse 失败无提示
30. **用户发截图问报错** — image 类型消息被忽略，无响应
31. **@ 在中间或末尾** — 只去掉开头第一个 @

#### 🟢 边界情况
32. **"delete daily abc"** — int() 转换非数字会 crash（无 try/except）
33. **"update xxx build_command" 不带 value** — 可能误设为空字符串
34. **短时间多条消息并行** — 日志可能交叉（task_id 隔离了但要注意）
