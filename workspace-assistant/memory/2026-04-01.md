### autobuild 代码架构备忘

改动时需要同时改的文件（铁律）：
1. **config.py** — `DEFAULT_DICT` 加新 key（bool 类型需要在这里注册，否则 update 时会当字符串处理）
2. **build_wheel.py** — 核心逻辑。build_wheel() 是编译、test_repo() 是测试
   - develop 阶段：test_repo 中约765行（hfai 提交 develop 命令）
   - build 阶段：build_wheel 中约1600行（hfai 提交编译命令）
   - 两处都有 `extra_command` 拼 hfai 参数
   - build 有 wheel_cache，test 现在有 test_cache
3. **utils.py** — `parse_show()` 展示配置、`parse_update()` 修改配置、`parse_build_command()` 解析指令
   - build_force/test_force 在 parse_build_command 开头预处理
   - parse_command() 是入口分发函数
4. **nlp_translate.py** — LLM 翻译 prompt
   - ⚠️ f-string 中花括号必须转义 `{{xxx}}`（铁律第16条）
   - VALID_COMMANDS 集合需要同步更新
   - FALLBACK_HELP 兜底帮助也需要更新
5. **feishu_api.py** — 调用 build_wheel/test_repo 的地方（两处：正常执行和 advance 执行）
6. **dict.py** — StateManager 持久化存储

### 关键变量引用链
- `command_dict[repo_name]` → repo 的所有配置（StateManager）
- `wheel_cache` / `test_cache` → 编译/测试结果缓存
- `config.DEFAULT_DICT` → 新仓库初始化时的默认值
- `config.OVERRIDABLE_KEYS` → 支持三层覆盖的 key（集群级/镜像级）
- `resolve_config()` → 三层覆盖查找
- `_safe_bool()` → 安全转 bool（兼容字符串 'True'/'False'）
